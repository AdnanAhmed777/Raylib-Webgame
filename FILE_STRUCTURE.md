## File Descriptions

### Core Files (YOU MAINTAIN THESE)

#### `snake_game.c`
- Your main game code in C
- Contains all game logic
- Uses Raylib for graphics
- No modification needed for web deployment!

#### `index.html`
- HTML wrapper for the game
- Provides the game canvas
- Loads the WebAssembly module
- Handles loading screen and UI

#### `README.md`
- Main project documentation
- Installation instructions
- Usage guide
- Credits and license

#### `QUICKSTART.md`
- Step-by-step deployment guide
- Troubleshooting tips
- Quick reference

### Build Files

#### `Makefile`
- Automates compilation
- Supports multiple targets (web/native)
- Cross-platform support
- Commands:
  - `make` or `make web` - Build for web
  - `make native` - Build desktop version
  - `make serve` - Run local web server
  - `make clean` - Remove build files

#### `build.sh`
- Shell script for building
- Alternative to Makefile
- Compiles C code to WebAssembly
- Creates docs/ folder

### GitHub Files

#### `.github/workflows/deploy.yml`
- GitHub Actions configuration
- Automatic build on push
- Deploys to GitHub Pages
- Runs on every commit to main branch

#### `.gitignore`
- Specifies files Git should ignore
- Excludes build artifacts
- Keeps repository clean
- Standard C project ignores

### Generated Files (AUTO-CREATED)

#### `docs/` folder
- Created by build process
- Contains web-ready files
- Deployed to GitHub Pages
- Don't edit these manually!

#### `docs/index.html`
- Final web page
- Includes embedded game
- Auto-generated from build

#### `docs/index.js`
- JavaScript glue code
- Generated by Emscripten
- Handles WebAssembly loading

#### `docs/index.wasm`
- Compiled WebAssembly binary
- Your C code compiled to web format
- Runs in browser

## Adding New Features

When you add new features to your game:

1. **Edit `snake_game.c`**
   - Add your code
   - Test locally if possible

2. **Test Build:**
   ```bash
   make web
   make serve
   ```

3. **Commit Changes:**
   ```bash
   git add snake_game.c
   git commit -m "Add new feature: description"
   git push
   ```

4. **Wait for Deployment:**
   - GitHub Actions will rebuild
   - Live in 2-3 minutes

## Troubleshooting File Issues

### Missing files after git clone?

```bash
# Check if all files are there
ls -la

# If missing .github folder
mkdir -p .github/workflows
# Then copy deploy.yml there
```

### Build files not updating?

```bash
# Clean and rebuild
make clean
make web
```

### GitHub Pages showing old version?

1. Clear browser cache
2. Wait 5 minutes for CDN update
3. Force refresh (Ctrl+F5)
4. Check GitHub Actions completed successfully
